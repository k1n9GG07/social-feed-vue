# Веб-приложение «Дневник» (Diary App)

Интерактивное веб-приложение для ведения личного дневника, разработанное на **Vue 3** с использованием **Pinia** и **Composition API**. Проект выполнен в стиле социальной сети (по макету Social Book) и поддерживает полноценную работу с постами и комментариями.

## 🚀 Основные функции

- **Лента постов**:
  - Создание новых записей с мгновенным добавлением в ленту.
  - Редактирование текста постов в модальном окне с возможностью отмены.
  - Удаление постов (с автоматическим удалением связанных комментариев).
- **Система комментариев**:
  - Просмотр комментариев к каждому посту (переключаемая видимость).
  - Добавление новых комментариев.
  - Удаление комментариев.
- **Сохранение данных**:
  - Все данные сохраняются в `localStorage`, что позволяет сохранять прогресс после перезагрузки страницы.
- **Валидация и надежность**:
  - Проверка на пустые поля при создании постов и комментариев.
  - Обработка ошибок при работе с хранилищем данных.

## 🛠 Технологический стек

- **Vue 3** (Composition API, Script Setup)
- **Pinia** (State Management)
- **TypeScript** (Типизация данных)
- **Vite** (Сборка проекта)
- **Vitest** (Unit и компонентное тестирование)
- **ESLint** (Линтинг и качество кода)
- **GitHub Actions** (Автоматический деплой на GitHub Pages)

## 📦 Установка и запуск

1. **Клонирование репозитория**:
   ```bash
   git clone <url-вашего-репозитория>
   cd diary-app
   ```

2. **Установка зависимостей**:
   ```bash
   npm install --legacy-peer-deps
   ```

3. **Запуск в режиме разработки**:
   ```bash
   npm run dev
   ```

4. **Сборка для продакшена**:
   ```bash
   npm run build
   ```

5. **Запуск тестов**:
   ```bash
   npm run test
   ```

## 🌍 Деплой

Проект настроен на автоматический деплой через **GitHub Actions**. При пуше в ветку `main` запускается рабочий процесс, который собирает приложение и публикует его в ветку `gh-pages`.
